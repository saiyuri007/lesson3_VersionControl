Highlight #1 Versioning

This is a method of file keeping that saves versions or modified files as separate files. Each version is typically given a different name or designation and then can be used to restore a project, file, image, or document to a previous save point by loading that version of the file. This can be done locally by a user by creating multiple saves on a hard drive, for example, document_1, document_2, etc., but this can become cluttered or confusing. You can also utilize a VCS or versioning control system and have it save several versions of a document for you. This option is usually much easier to use and often has several benefits described in the next section.

Highlight #2 Version Control System

Also known as a VCS, revision control service (RCS), or source control management (SCM). This is a service or system that will control versioning for you. There are several benefits to using this service including the ability to work more effectively with a team. VCSâ€™s allow you to check a document or project out in order to make changes, commit changes, and ensure that a previous version is saved in case of an emergency, crash, or changes made that were not approved. Examples of these programs include SharePoint by Microsoft, which allows for full site management with versioning settings, PerForce, and GIT.

Highlight #3 GIT

Created by Linux kernel in 2005 in response to the breakdown of BitKeeper. GIT is a specific versioning system that is used by many major corporations such as Google, Netflix, and Microsoft. It is a DVCS or distributed versioning control system, which means that each person has a full copy of all the files on their local computer, and that there is no single point of failure in the event of a crash. Most VCS programs save versions as a map of changes to the document, whereas GIT saves snapshots of the versions and instead creates a link to the previous version of a file instead of creating an entirely new saved version of the file.

Highlight #4 Three States of GIT

Git has three stages in which your data can be stored. These are known as: staged, modified, and committed. Staged means you have marked the file that has been modified and it is awaiting its snapshot. You can have several staged versions of a file awaiting committal. Modified means that the document or project has been changed, but not saved to the database. Committed means that a document has been stored to the local database in which you choose to save these files. Even though you would typically first modify a file, then stage it, and finally commit it, it is possible to skip steps and go straight to committing a file.

Highlight #5 GIT Commands

Git runs very similarly to a command prompt on your PC. Git does not automatically track or ignore changes to documents, but you must tell it to do so using commands. Each command tells Git to perform a specific action. For example, if you want to see which files have been modified, then you would type git status into Git, and the system will run the command and display the desired information. Other commands include git add, which is a multi-purpose command that essentially says whatever you mark with this command will be committed in the next commit function, and .gitignore, which will tell git to ignore files meeting certain described patterns.
